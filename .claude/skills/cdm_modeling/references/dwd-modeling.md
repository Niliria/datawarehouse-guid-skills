# DWD 明细事实建模

根据 DWD spec、总线矩阵和 DIM 设计生成原子明细事实表。保持业务过程原子粒度，只获取 DIM 代理键，不展开维度描述属性。

## 设计步骤

1. 为每个业务过程生成一张 DWD。
2. 保留上游 `DWD表名`；缺失时才生成 `dwd_{domain}_{business_process}_{di|df}`。
3. 使用 `字段角色=grain_key` 构造粒度键；缺失时读取业务过程清单的 `粒度键`。
4. 使用总线矩阵正向标记确定维度集合。
5. 使用 DWD `fk` 与 DIM `pk/bk` 精确匹配具体字段。
6. 优先使用显式 `关联DIM表/关联DIM业务键` 消除歧义。
7. 将维度业务键转换为 `{entity}_sk`，不要展开 DIM 属性。
8. 保留退化维度、低基数属性、业务时间和度量。
9. 保留来源 ODS 表、字段和 join spec。

## 字段角色

| 字段角色 | DWD 处理方式 |
|---|---|
| `grain_key` | 作为事实行粒度键，支持联合粒度 |
| `fk` | 映射到 DIM 代理键 |
| `degenerate_dim` | 作为订单号、凭证号等退化维度保留 |
| `low_card_attr` | 作为必要状态或低基数明细属性保留 |
| `measure` | 保留来源、类型、可加性和聚合建议 |
| `business_time` | 作为业务事件时间保留 |

DWD 默认不保留用户名、手机号、门店名、商品名等 DIM 描述属性，也不生成宽表、汇总指标或应用口径。

## 维度关系解析

按以下优先级建立关系：

1. 使用显式 `关联DIM表/关联DIM业务键`。
2. 使用总线矩阵确认事实应关联的 DIM。
3. 将 DWD FK 字段名或来源字段与 DIM 业务键精确匹配。
4. 使用 `{entity}_id` 形式辅助匹配。
5. 匹配失败或并列时记录告警，不静默选择无依据关系。

ETL 关联形式：

```sql
LEFT JOIN dim_xxx d
  ON source.<来源FK字段> = d.<DIM业务键>
 AND d.pt = '${bizdate}'
```

关联 SCD Type II 当前版本时增加 `d.is_active = 1`。

## 事实类型

| 类型 | 适用场景 |
|---|---|
| `transaction` | 一行一笔业务事件 |
| `periodic_snapshot` | 固定周期状态截面 |
| `accumulating_snapshot` | 一行跟踪完整生命周期 |
| `factless` | 无数值度量的行为事实 |

非 `factless` 事实表必须至少包含一个度量。

## 度量规则

- 金额、数量、次数通常使用 `SUM`。
- 比率和单价通常不可直接求和。
- 余额、库存通常为半可加度量，不得跨时间直接求和。
- 使用 DWD spec 的 `度量类型` 和 `聚合建议`，不要根据字段名覆盖上游口径。
- 保留 `是否派生` 和 `派生逻辑` 供后续层审查；不要在 DWD 模板中重新创造派生指标。

## 设计字典

| 字段 | 说明 |
|---|---|
| `table_name` | DWD 表名 |
| `domain` | 主题域编码 |
| `business_process` | 业务过程代码 |
| `grain` | 粒度声明 |
| `grain_keys` | 粒度键列表 |
| `fact_type` | 事实类型 |
| `dimension_refs` | DIM、DWD FK 与业务键映射 |
| `detail_fields` | 退化维度、低基数属性、业务时间 |
| `measures` | 度量列表 |
| `source_tables` | 来源 ODS 表 |
| `source_joins` | ODS 关联关系 |

## 强制规则

1. 保持一张 DWD 对应一个明确业务过程和一个稳定粒度。
2. 使用 `pt STRING` 分区。
3. 保留 `etl_insert_time`、`etl_update_time` 和 `source_system`。
4. 使用 ORC 和 SNAPPY 作为默认存储设置。
5. 不自动补充日期维度；仅使用总线矩阵或显式映射定义的维度。
6. 不在 SQL 模板中推断业务语义。

## 验收标准

1. 确认每个 DWD 有明确粒度和粒度键。
2. 确认每个非 `factless` DWD 至少有一个度量。
3. 确认总线矩阵每个正向维度均映射到 DIM 和 DWD FK。
4. 确认全部度量存在字段名、来源字段、类型和聚合建议。
5. 确认退化维度、低基数属性和业务时间未被丢弃。
6. 确认 DDL/ETL 不包含未渲染占位符。

## 常见反模式

- 粒度不明确：回到业务过程清单修正。
- 度量硬编码：改为消费 DWD spec。
- 维度指向不存在的 DIM：修正矩阵或 DIM spec。
- 大量 DIM 描述属性进入 DWD：移回 DIM 或下沉 DWS/ADS。
- 半可加度量跨时间求和：按期末、平均或业务指定逻辑处理。
- 模板自行猜测字段：将业务语义移回设计上下文。
