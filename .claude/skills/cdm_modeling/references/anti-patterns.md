# CDM 建模反模式

本文件列出生成 DIM/DWD/CDM 产物时必须避免的设计问题。发现反模式时，应写入 `validation_report.md` 或回到上游文档修正。

## 1. 输入反模式

### 上游文档缺少粒度

问题：DWD 表无法判断一行代表什么业务事实。

处理：回到总线矩阵文档补充 `grain`。

### 维度和度量混在备注里

问题：解析不稳定，容易误把文本说明当字段。

处理：使用结构化 `dimensions` 和 `measures`。

### ODS 字段没有分类

问题：DIM 属性、外键、度量都只能靠命名猜测。

处理：上游 ODS 元数据解析文档补充 `classification` 和 `dimension`。

## 2. DIM 反模式

### 缺少业务键

问题：无法与源系统对账，SCD JOIN 不稳定。

处理：补充明确业务键；不得只依赖代理键。

### 混合粒度

问题：同一 DIM 同时存客户、订单、账户等不同粒度。

处理：拆分为不同维度或回到上游定义粒度。

### 过度规范化

问题：DIM 只有外键，查询必须多层 JOIN，失去维度表价值。

处理：在 DIM 中反范式化常用描述属性。

### SCD 滥用

问题：所有维度默认 Type II，导致存储膨胀和查询复杂。

处理：根据业务历史追踪需求和源表时间字段选择 SCD。

### 日期维度使用 Type II

问题：日期维度静态，不应拉链。

处理：日期/时间维度使用 Type I。

## 3. DWD 反模式

### 没有明确粒度

问题：度量无法正确聚合，事实表不可解释。

处理：补充 `grain`，并在必要时做唯一性校验。

### 度量硬编码

问题：所有事实表都有 `quantity`、`amount`，与业务不匹配。

处理：度量必须来自总线矩阵文档。

### 维度外键指向不存在的 DIM

问题：ETL JOIN 失败，星形模型不完整。

处理：补齐 DIM 或移除不成立的维度引用。

### 把大量描述属性塞进 DWD

问题：事实表膨胀，维度口径难统一。

处理：描述属性进入 DIM；DWD 只保留必要退化维度和外键。

### 半可加度量跨时间求和

问题：余额、库存等指标聚合错误。

处理：标记半可加，跨时间使用期末、平均或其他业务指定逻辑。

## 4. ETL 反模式

### 代理键每日重排

问题：历史事实外键失效。

处理：代理键生成必须稳定；Type II 新版本生成新 SK，旧 SK 不变。

### 模板中推断业务语义

问题：SQL 模板不可维护，业务规则分散。

处理：业务语义必须在设计字典完成，模板只渲染。

### 源字段缺失仍生成 SQL

问题：SQL 看似完整但无法执行。

处理：在字段映射和校验报告中标记缺失。

## 5. 文档反模式

### 文档承诺不存在的产物

问题：用户按文档执行会失败。

处理：文档输出契约必须与代码生成产物一致。

### 规则文件存在但不生效

问题：用户误以为 YAML 已驱动生成逻辑。

处理：未接入代码的规则应作为 reference 文档或 legacy 资料。

### 示例不能一条命令跑通

问题：skill 无法验证。

处理：维护 `examples/basic/` 和固定验证命令。
