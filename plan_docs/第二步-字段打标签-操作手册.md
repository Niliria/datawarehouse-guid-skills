## 二、标注流程（单表）

### 2.1 步骤 1：查询表结构

```sql
-- 查询表的所有字段（含注释）
SELECT
    column_name AS '字段名',
    data_type AS '数据类型',
    character_maximum_length AS '长度',
    is_nullable AS '是否可空',
    column_default AS '默认值',
    column_comment AS '注释'
FROM information_schema.columns
WHERE table_schema = 'yummy'
  AND table_name = '表名'
ORDER BY ordinal_position;
```

**执行结果示例：**
```
+------------------+-------------+--------+-----------+---------------+-----------------+
| 字段名           | 数据类型    | 长度   | 是否可空  | 默认值        | 注释            |
+------------------+-------------+--------+-----------+---------------+-----------------+
| id               | int         | NULL   | NO        | NULL          | 主键            |
| evaluation_id    | varchar     | 64     | NO        | NULL          | 评价ID          |
| supplier_id      | varchar     | 64     | YES       | NULL          | 供应商ID        |
| score            | decimal     | NULL   | YES       | NULL          | 评分            |
| create_time      | datetime    | NULL   | YES       | NULL          | 创建时间        |
+------------------+-------------+--------+-----------+---------------+-----------------+
```

### 2.2 步骤 2：逐字段标注

对照 **判定规则速查表**，为每个字段标注标签。

**标注模板（Markdown 格式）：**

```markdown
# 表：technical_evaluation

| 序号 | 字段名 | 数据类型 | 标签 | 关联实体/说明 |
|------|--------|---------|------|---------------|
| 1 | id | int | [代理键] | 自增主键 |
| 2 | evaluation_id | varchar(64) | [业务键] | 评价主键 |

| 5 | evaluator_id | varchar(64) | [外键] | 引用 ods_itf_po_employee_df.employee_id |
| 6 | score | decimal(5,2) | [可加度量] | 评分（0-100） |

| 8 | evaluation_date | datetime | [业务时间] | 评价时间 |
```

**标注模板（Excel 格式）：**

| 序号 | 字段名 | 数据类型 | 标签1 | 标签2 | 标签3 | 关联实体 | 说明 | 备注 |
|------|--------|---------|-------|-------|-------|----------|------|------|
| 1 | id | int | [代理键] | | | | 自增主键 | |
| 2 | evaluation_id | varchar(64) | [业务键] | | | 评价 | 评价主键 | |
| 3 | supplier_id | varchar(64) | [外键] | | | 供应商 | 引用 supplier_evaluation_list | |

**标签列说明：**
- 最多 3 个标签列（因为一个字段可能有多个标签，如 [业务键][外键]）
- 如果只有 1 个标签，填在标签1列
- 关联实体：填写该字段对应的业务实体（如用户、商品、订单）
- 说明：补充标注理由或特殊说明

### 2.3 步骤 3：验证标注

使用 **标注检查清单** 自查：

- [ ] 所有 ID/编号类字段是否都标注了 [业务键] 或 [代理键]？
- [ ] 所有引用其他表的字段是否都标注了 [外键] 并注明引用表？
- [ ] 所有数值字段是否都标注了度量标签（可加/半可加/不可加）？
- [ ] 所有时间字段是否区分了 [业务时间] 和 [技术时间]？
- [ ] 所有文本描述字段是否标注了 [维度属性] 或 [退化维度]？
- [ ] 所有系统字段是否标注了 [技术字段]？
- [ ] 低基数枚举字段是否考虑了 [退化维度]？
- [ ] 是否存在字段未标注任何标签？（漏标）
- [ ] 是否存在字段标注了矛盾的标签？（如同时标 [业务键] 和 [技术字段]）

## 五、常见问题与解决方案

### 5.1 问题1：字段注释为空

**现象：** `column_comment` 为空，无法判断业务含义。

**解决方案：**
1. 查看字段名猜测业务含义（如 `supplier_id` → 供应商ID）
2. 采样数据查看实际值：
   ```sql
   SELECT supplier_id, COUNT(*) FROM technical_evaluation GROUP BY supplier_id LIMIT 10;
   ```
3. 询问业务方或查看源系统文档
4. 暂时标注为 [待确认]，后续补充

---

### 5.2 问题2：外键引用表不明确

**现象：** `supplier_id` 可能引用 `supplier_evaluation_list`，也可能引用其他供应商表。

**解决方案：**
1. 查询所有含 `supplier_id` 的表：
   ```sql
   SELECT table_name, column_name
   FROM information_schema.columns
   WHERE table_schema = 'yummy'
     AND column_name = 'supplier_id';
   ```
2. 检查哪张表的 `supplier_id` 是主键：
   ```sql
   -- 查询 supplier_evaluation_list 的主键
   SELECT column_name
   FROM information_schema.KEY_COLUMN_USAGE
   WHERE table_schema = 'yummy'
     AND table_name = 'supplier_evaluation_list'
     AND constraint_name = 'PRIMARY';
   ```
3. 如果多张表都有 `supplier_id` 且都是主键 → 标注为 [外键]（引用多个表候选），第三步再确认

---

### 5.3 问题3：数值字段是否为度量不确定

**现象：** `score` 是评分，不确定是否为度量。

**解决方案：**
1. 查看数据范围：
   ```sql
   SELECT MIN(score), MAX(score), AVG(score) FROM technical_evaluation;
   ```
2. 判断业务含义：
   - 如果是"评分"（0-100）→ [不可加度量]（不能把多个评分相加）
   - 如果是"扣分"（可累计）→ [可加度量]
   - 如果是"平均分"（已计算过）→ [不可加度量]

---

### 5.4 问题4：时间字段是否为业务时间不确定

**现象：** `create_time` 是创建时间，但不确定是否为业务关心的时间。

**解决方案：**
1. 查看字段注释（如果有）
2. 采样数据看时间分布：
   ```sql
   SELECT DATE(create_time), COUNT(*) FROM technical_evaluation GROUP BY DATE(create_time) ORDER BY DATE(create_time) DESC LIMIT 10;
   ```
3. 询问业务方：这个时间是否用于分析业务趋势？
4. 默认策略：除非明确是系统时间（如 `update_time`），否则标为 [业务时间]

### 7.2 快速 SQL 命令

```sql
-- 1. 查询某张表的所有字段
SELECT column_name, data_type, column_comment
FROM information_schema.columns
WHERE table_schema = 'yummy' AND table_name = 'technical_evaluation'
ORDER BY ordinal_position;

-- 2. 查询所有含某字段的表
SELECT table_name, column_name
FROM information_schema.columns
WHERE table_schema = 'yummy' AND column_name = 'supplier_id';

-- 3. 查询某张表的主键
SELECT column_name
FROM information_schema.KEY_COLUMN_USAGE
WHERE table_schema = 'yummy'
  AND table_name = 'supplier_evaluation_list'
  AND constraint_name = 'PRIMARY';

-- 4. 统计某字段的唯一值数量（判断基数）
SELECT COUNT(DISTINCT evaluation_type) FROM technical_evaluation;

-- 5. 验证外键关系
SELECT COUNT(*) AS not_found
FROM technical_evaluation a
LEFT JOIN supplier_evaluation_list b ON a.supplier_id = b.supplier_id
WHERE b.supplier_id IS NULL;
```
